### 2. 移动端形态

**实现方式：**
- **单代码库 + Next.js 响应式**
- **Layout 独立组件**：移动端独立的 layout（金刚区 + 全屏 chat），PC 保持现有（sidebar + 右侧 chat 栏）
- 业务页面尽量复用，仅 layout 和登录页是两套

**移动端登录页（简化版）：**
- Hero 标题保留
- 双角色卡片**垂直堆叠**（不是横向）
- 项目亮点区**精简版**（三段都展开，但每段只 1 行核心句）：
  ```
  🎬 VibeCoding × Spec Coding
     AI 全程编码 + 文档约束生成，避免决策漂移

  🧬 Palantir Ontology 落地
     业务对象 / 关系 / Action 在底层显式建模

  🤝 Copilot 人机协同
     界面全 API 化，AI 直接调接口而不是模拟点击

  公众号「pmYangKun」全程记录
  ```
- **GitHub 链接去掉**（手机不 clone）

**金刚区主导航（5 入口，中间凸起）：**

```
                    [Chat 主屏]
                    
       线索   客户   [💬]   跟进   我的
                    ↑凸起
```

| 入口 | 内容 |
|------|------|
| 线索 | Lead 列表（按 DataScope 自动过滤） |
| 客户 | Customer 列表 |
| 💬 AI 助手 | **凸起按钮**，全屏 Chat 模式 |
| 跟进 | 最近跟进流 + 录入入口 |
| 我的 | 账号信息 + **一键切换角色** + 退出 |

**关键约束：**
- **登录后默认进入 Chat 全屏**（点中间凸起按钮的状态）
- "我的" tab 必须支持**一键角色切换**（sales01 ↔ manager01），无需重新输密码

**移动端 Copilot 写类操作交互（chat 内嵌卡片范式）：**

PC 端的"chat 右侧 + form 左侧"双栏模式在移动端无法照搬（全屏 chat 没有可引导的左侧表单）。移动端写类操作（创建客户/联系人/跟进等）采用 **chat 内嵌卡片** 范式，对标 ChatGPT canvas / Claude artifact：

- AI 完成预填后，在 chat 流里**渲染表单预览卡片**，每个对象（客户、联系人、跟进）一张卡
- 卡片显示：对象类型 + 关键字段预览（如"客户：深圳前海微链 / 联系人：赵鹏 / 跟进：拜访合作方案"）+ 状态徽章
- **点卡片** → 弹**底部抽屉（bottom sheet）**展示完整可编辑表单
- 抽屉里点"确认提交" → 关闭抽屉 → 卡片状态变 ✅ 已创建，并显示创建成功的对象 ID/链接
- 多对象批量创建 → 多张卡片在 chat 里**纵向堆叠**，状态各自独立
- 用户可以继续在 chat 追问"再补一条备注" → AI 再追加一张卡片

**为什么选这个范式：**
- 不打断 chat 上下文（用户能持续追问）
- 多对象天然叠多张卡片，比"全屏路由跳 3 次"或"sheet 串行弹"体验好
- 移动端访客大概率没用过 SFA CRM 但用过 ChatGPT，看一眼就懂
- 这个交互本身就是"Native AI CRM"亮点的可视化证据

**新增组件（spec-kit /plan 阶段需识别）：**
- `<ChatFormCard>`：chat 流内的表单预览卡片，含 pending/editing/submitted 三态
- `<MobileFormSheet>`：底部抽屉表单，复用现有表单字段组件
- AI 工具调用层：写类 tool 返回结构化"待确认对象"而非直接写库，由前端卡片驱动用户确认后再 commit

### 3. 演示引导（PC + 移动）

**引导卡片机制：**

- 卡片点击 → 问题自动发送到 Chat 并执行（**一键发送**，零摩擦）
- 实现使用**现有的 sessionStorage 模式**（chat 导航预填已经在用，延用）
- **永远显示**（demo 性质，没有"老用户"概念，每次进入 chat 都看到）

**PC 端引导位置：方案 B**

- 登录后默认进 **Dashboard**（现有页面）
- Dashboard 顶部加"AI 引导区"：欢迎语 + 卡片网格 + 切换角色提示
- 下方保留正常 Dashboard 数据

**移动端引导位置：**

- 登录后默认进 Chat 全屏
- Chat 首屏顶部显示推荐问题卡片（无消息时显示，有消息后折叠或消失）

**卡片视觉格式（参考）：**

```
┌─────────────────────────────────────────┐
│  ⚡ 一句话搞定多个动作                    │
│  "我今天拜访了深圳前海微链的赵鹏，         │
│   聊了合作方案，对方很感兴趣。            │
│   帮我把客户、联系人、跟进都录进去。"      │
│  [试试看 →]                             │
└─────────────────────────────────────────┘
```

短标题 + 完整问题预览 + 一键发送按钮。

> **文案原则：** 给访客（B 端 PM，但不是 CRM 用户）看的话术必须把"AI 干了什么"具体化——避免"该做的都做了"这种行话，要把对象点出来（"客户、联系人、跟进"）。`src/demo/copilot-cases.md` 案例 3 原话已同步更新。

**"切换角色"卡片：**
- 点击 → 弹确认框 → 退出当前账号 → 自动登录目标账号 → 进入对应引导
- 后端需要新增"快速切换"API（或者前端调登录 API + 当前 logout API）

**"谁在偷懒"案例呈现（方案 A 文字加强版）：**
- AI 回答用 markdown 表格 + 关键数字加粗 + emoji（🔴⚠️✅）
- 每个销售一行：姓名 + 跟进活跃度评级 + 风险线索数
- 不做富组件嵌入（B 方案推后到主题攻势"对话式 CRM"再做）
