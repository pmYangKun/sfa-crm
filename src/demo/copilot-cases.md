# SFA CRM Copilot 演示案例集

> 直接复制粘贴到 Copilot 对话框即可演示。每条标注了预期行为，方便讲解。

---

## 一、查询类（AI 直接执行，无需人工确认）

### 1.1 简单条件查询

```
帮我看看华北有哪些线索
```

> **预期：** AI 调用搜索接口，返回线索列表 + 自动归纳区域分布、来源构成等洞察。

### 1.2 多条件组合查询

```
华南大区，最近7天没人跟进的线索有哪些？
```

> **预期：** AI 组合 region + last_followup_at 两个筛选条件，返回结果并提示"这些线索有自动释放风险"。

### 1.3 跨对象关联查询

```
李明这个联系人关联了哪些企业？
```

> **预期：** AI 搜索 Contact，找到关联的 Lead/Customer，如果存在 ContactRelation 还会展示跨企业人脉关系。

### 1.4 统计分析类查询

```
这个月我的团队一共转化了多少客户？转化率多少？
```

> **预期：** AI 查询 Lead(converted) + Lead(total)，计算转化率，按销售分列。主管视角下展示团队数据。

### 1.5 客户状态查询

```
我名下有哪些客户快到大课转化窗口期了？
```

> **预期：** AI 查询 Customer，计算 `now - created_at`，筛选出 7-14 天内的客户，提示窗口剩余天数。

### 1.6 公共池查询

```
现在公共池里有多少条线索？哪个大区最多？
```

> **预期：** AI 查询 pool=public 的线索，按大区分组统计，给出分布概况。

### 1.7 模糊查询 + 业务判断

```
之前有个做PCB的公司，好像叫嘉什么创，帮我找找
```

> **预期：** AI 用模糊匹配搜索公司名，返回候选结果（如"嘉立创"），展示归属、阶段等关键信息。

## 二、录入类（AI 导航到预填表单，人工确认提交）

### 2.1 新建线索 — 基础信息

```
新建一条线索，公司叫华为技术有限公司，在华南，来源是转介绍
```

> **预期：** AI 生成"→ 去创建线索"导航按钮，跳转后表单已预填公司名、大区、来源。用户确认后提交。

### 2.2 新建线索 — 从自然语言推断字段

```
老李介绍了一家深圳做芯片的公司给我，叫芯源微电子
```

> **预期：** AI 推断来源="转介绍"、大区="华南"（深圳），生成导航按钮，表单预填三个字段。

### 2.3 录入跟进记录

```
我今天下午去拜访了深圳前海微链的赵鹏，聊了合作方案，对方很感兴趣，约了下周二再谈细节
```

> **预期：** AI 搜索线索 → 确认联系人 → 生成"→ 录入跟进记录"导航，预填类型=拜访、联系人=赵鹏、内容摘要。

### 2.4 录入关键事件 — 送书

```
今天给北京数字颗粒科技的张总寄了一本《决胜B端》
```

> **预期：** AI 搜索线索 → 确认联系人 → 生成"→ 记录送书事件"导航，预填事件类型=book_sent、contact=张总、sent_at=今天。

### 2.5 添加联系人

```
前海微链那边除了赵鹏，还有个技术负责人叫刘洋，微信号 liuyang2026
```

> **预期：** AI 搜索线索 → 生成"→ 添加联系人"导航，预填姓名、微信号。如果微信号已存在于其他企业，AI 会提示"发现重复联系人，可能存在关联关系"。

## 三、多步决策类（AI 拆解任务 + 业务建议）

### 3.1 拜访后一句话处理全部

```
我今天拜访了深圳前海微链的赵鹏，聊了合作方案，对方很感兴趣。帮我把该做的都做了。
```

> **预期：** AI 搜索线索 → 确认联系人是 KP → 查看历史跟进 → 生成两个导航操作：
> 1. 录入跟进记录（预填拜访内容）
> 2. 创建关键事件-拜访KP（预填联系人）
> 并给出建议：记录合作方案内容、对方反馈、下一步计划。

### 3.2 线索状态评估 + 建议

```
北京数字颗粒科技最近情况怎么样？我该怎么跟？
```

> **预期：** AI 查线索详情 + 全部跟进记录 + 关键事件，综合分析后给出判断：
> - 上次跟进距今 X 天（是否接近释放阈值）
> - 送书后是否有回应
> - 建议下一步动作（打电话/约拜访/送课程邀请）

### 3.3 批量跟进提醒

```
帮我看看我名下有哪些线索快要被自动释放了，我该优先跟哪几个？
```

> **预期：** AI 查询私有池中 last_followup_at 接近 10 天阈值的线索，按紧急程度排序，给出跟进优先级建议和理由。

### 3.4 转化决策辅助

```
天津智联云的小课款已经到了，帮我走转化流程
```

> **预期：** AI 确认线索信息 → 检查是否已有付款记录 → 生成"→ 确认转化"导航。同时提示：转化后将创建客户记录，联系人和跟进历史自动迁移，14天大课转化窗口开始计时。

### 3.5 线索冲突处理建议

```
我想录一个新线索叫嘉为科技，但系统说跟已有的重复了，怎么办？
```

> **预期：** AI 解释唯一性检测结果（精确匹配 or 模糊预警），展示已有线索的归属人和状态，建议：如果确实是同一家→联系归属人或主管；如果是不同公司→补充组织机构代码区分。

## 四、管理操作类（主管/Admin 视角）

### 4.1 分配线索

```
把公共池里北京数字颗粒科技分配给销售01
```

> **预期：** AI 检查销售01 的私有池容量 → 生成"→ 分配线索"导航，预填目标销售。如果私有池已满，AI 会提示"销售01 私有池已有 98 条，接近上限 100"。

### 4.2 团队线索健康度检查

```
看看我团队里谁的线索跟进最不积极？
```

> **预期：** AI 查询团队所有销售的线索跟进情况，按"平均跟进间隔天数"排序，指出谁的线索最久没跟进、有多少条快要被自动释放。

### 4.3 释放线索

```
销售03上周离职了，他名下的线索全部释放回公共池
```

> **预期：** AI 查询销售03 名下所有 active 线索数量 → 提示"共 X 条线索将被释放" → 生成批量释放的导航操作。

### 4.4 客户调配

```
把客户"太古可口可乐"从销售01调给销售05
```

> **预期：** AI 搜索客户 → 生成"→ 调配客户"导航，预填新归属人。提示：调配后销售01 将立即失去该客户的可见性。

### 4.5 团队日报检查

```
今天谁还没交日报？
```

> **预期：** AI 查询团队日报提交情况，列出未提交名单和对应的跟进记录数（有跟进但没交日报 vs 一整天没跟进）。

## 五、日报类

### 5.1 查看日报草稿

```
帮我看看今天的日报草稿
```

> **预期：** AI 调取系统自动生成的草稿（基于当天跟进记录汇总），展示内容，可以直接提交或补充修改。

### 5.2 一句话补充并提交

```
今天日报再加一条：下午跟赵鹏确认了下周二的拜访时间。然后直接提交给队长。
```

> **预期：** AI 在草稿基础上追加内容 → 生成"→ 提交日报"导航，提交对象预填队长。

---

## 六、系统配置类（Admin 视角）

### 6.1 调整业务规则

```
华东大区要求线索只能在战队内抢占，帮我改一下配置
```

> **预期：** AI 识别这是大区抢占规则变更 → 生成"→ 修改系统配置"导航，展示当前华东的规则和将要改成的 same_region_only 模式。

### 6.2 查询当前配置

```
现在自动释放的规则是什么？多久没跟进会被释放？
```

> **预期：** AI 查询 SystemConfig，展示 followup_release_days=10、conversion_release_days=30，并解释两条规则的触发条件。

---

## 七、AI 洞察类（超越传统 CRM 的能力）

### 7.1 基于跟进历史的策略建议

```
赵鹏已经聊了三次都没推进到报价，你觉得问题出在哪？
```

> **预期：** AI 读取全部跟进记录，分析对话脉络，给出可能的卡点判断（决策链不完整/预算未确认/竞品干扰等），建议下一步策略。

### 7.2 跨客户模式识别

```
最近转化成功的客户有什么共同特征？
```

> **预期：** AI 查询近期 converted 的线索，归纳共性（来源渠道、跟进次数、关键事件类型、转化周期），给出数据驱动的洞察。

### 7.3 人脉网络利用

```
我想拓展华北的新客户，有没有现有客户的联系人可以帮忙转介绍？
```

> **预期：** AI 查询 ContactRelation，找到华北客户/线索的联系人关系网络，识别可能的转介绍路径。

---

## 使用说明

- **查询类**（第一、七类）：AI 直接执行并返回结果，无需人工操作
- **写入类**（第二至六类）：AI 生成导航按钮 + 预填表单，用户确认后提交
- 每个案例可根据实际演示数据调整公司名、人名等细节
- 建议演示顺序：1.1 → 2.1 → 3.1 → 7.1，从简单到复杂，最后展示 AI 洞察能力
